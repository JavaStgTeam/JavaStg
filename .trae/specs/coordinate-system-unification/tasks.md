# 坐标系统一化项目 - 实现计划

## [x] 任务1: 修复玩家类的边界检测逻辑
- **优先级**: P0
- **依赖**: None
- **描述**: 
  - 修改Player类中的边界检测逻辑，使用游戏逻辑坐标系的固定边界
  - 替换基于实际画布尺寸的边界计算，使用CoordinateSystem提供的边界方法
- **验收标准**: AC-1, AC-4
- **测试要求**:
  - `programmatic` TR-1.1: 玩家移动到边界时被正确限制在游戏逻辑坐标系内
  - `programmatic` TR-1.2: 玩家出生位置正确，在屏幕底部上方40像素处
- **注意**: 确保使用CoordinateSystem的getLeftBound()、getRightBound()、getTopBound()、getBottomBound()方法获取边界
