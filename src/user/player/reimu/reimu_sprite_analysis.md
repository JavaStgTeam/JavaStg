## 技术使用建议

### 精灵表渲染参数
- **精灵表宽度**: 384像素
- **精灵表高度**: 272像素
- **角色精灵尺寸**: 48x48像素
- **子弹精灵尺寸**: 32x32像素
- **特效精灵尺寸**: 64x32像素
- **框架精灵尺寸**: 64x64像素
- **角色动画帧数**: 8帧/方向
- **角色动画行数**: 3行 (站立/左移/右移)

### 动画实现
1. **站立动画**: 使用精灵ID 1-8，按顺序循环
2. **左移动画**: 使用精灵ID 9-16，按顺序循环
3. **右移动画**: 使用精灵ID 17-24，按顺序循环
4. **动画速度**: 建议每5-10帧切换一次精灵

### 子弹和特效使用
- **蓝色子弹**: 精灵ID 25-26
- **红色子弹**: 精灵ID 27-30, 33-36
- **子弹特效**: 精灵ID 31-32, 37-38
- **边框特效**: 精灵ID 39-42

## 坐标计算方法

### 角色精灵坐标计算
```java
int spriteX = (frameIndex % 8) * 48;
int spriteY = animationType * 48;
// animationType: 0=站立, 1=左移, 2=右移
// frameIndex: 0-7 (8帧循环)
```

### 子弹精灵坐标计算
```java
// 第一行子弹 (y=144)
int bulletX1 = (bulletIndex % 6) * 32;
int bulletY1 = 144;

// 第二行子弹 (y=176)
int bulletX2 = (bulletIndex % 6) * 32;
int bulletY2 = 176;
```

### 框架精灵坐标计算
```java
int frameX = (frameIndex % 4) * 64;
int frameY = 208;
// frameIndex: 0-3
```

## 资源使用说明
- **纹理加载**: 使用384x272的完整精灵表
- **内存优化**: 精灵表只加载一次，通过坐标裁剪使用
- **性能建议**: 使用精灵表批处理渲染，减少绘制调用

## 兼容性
- 精灵表尺寸和布局与现有__ReimuPlayer类完全兼容
- 可直接用于SpriteSheetRenderer.drawSpriteFrame方法
